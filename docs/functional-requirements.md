# Functional Requirements

## Task Management

1. **Add a Task** — The user can create a new task by providing a title.
2. **Edit a Task** — The user can edit the title or details of an existing task.
3. **Delete a Task** — The user can remove a task from the list.
4. **Mark a Task as Complete** — The user can mark a task as done, and it will be visually distinguished from incomplete tasks.

## Due Dates

5. **Add a Due Date** — The user can assign a due date to a task.
6. **Edit a Due Date** — The user can update or remove the due date of an existing task.
7. **Overdue Indicator** — Tasks whose due date has passed and are not yet complete are highlighted as overdue.

## Sorting & Ordering

8. **Sort by Due Date** — Tasks can be sorted in ascending order by due date (earliest first).
9. **Sort by Creation Date** — Tasks can be sorted by the date they were created.
10. **Sort by Completion Status** — Incomplete tasks appear before completed tasks by default.

## Filtering

11. **Filter by Status** — The user can filter tasks to show all, only active, or only completed tasks.
12. **Filter by Due Date** — The user can filter tasks to show only tasks due today, this week, or overdue.

## Persistence

13. **Save Tasks** — Tasks are persisted so they are available after the page is refreshed.
